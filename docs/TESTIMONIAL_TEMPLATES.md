## ⏰ **When to Ask**

### **Best Moments to Request Testimonials**

```
🎉 After a Win
• User ships their project
• User gets funding
• User lands big client
• User reports time saved

📧 After Positive Support Interaction
• User thanks you for help
• Issue resolved quickly
• User expresses satisfaction

📊 After Milestone Usage
• User generates 100+ modules
• User completes 5+ projects
• 30-day check-in (if happy)

🎓 After Course/Tutorial Completion
• User finishes video series
• User completes documentation
• User joins Discord and participates
```

### **Timing Guidelines**

```
✅ DO ask within 24-48 hours of positive event
✅ DO ask when enthusiasm is high
✅ DO make it easy (provide templates)
✅ DO offer incentives (optional)

❌ DON'T ask immediately after purchase
❌ DON'T ask if user had support issues
❌ DON'T guilt trip if they say no
❌ DON'T fake testimonials (ever!)
```

## 🎤 **Interview Questions**

### **For Written Testimonials**

```
Basic Questions (for short quotes):

1. What were you building when you found Backend Starter Kit?

2. How did you build APIs before using this?

3. What was your biggest frustration before?

4. What's the #1 thing you love about the kit?

5. How much time do you think it saved you?

6. Would you recommend it? To whom?
```

### **For Case Studies (In-Depth)**

```
Background & Context:

1. Tell me about yourself (role, company, experience level)

2. What project were you working on?

3. What was your timeline/deadline?

4. What was your budget (time or money)?

The Problem:

5. How were you building APIs before?

6. What was frustrating about that approach?

7. What were you considering as alternatives?

8. What almost stopped you from buying?

The Solution:

9. What made you decide to try Backend Starter Kit?

10. How long did setup take?

11. What was your first impression?

12. Which features did you use most?

The Results:

13. How long did it take to build your project?

14. How does that compare to your usual timeline?

15. What specific time savings did you notice?

16. Any bugs/security issues prevented?

17. What was the outcome? (launched, raised funding, etc.)

The Experience:

18. How was the documentation?

19. Did you need support? How was it?

20. What surprised you (positively)?

21. What could be better?

Recommendation:

22. Who would benefit most from this?

23. Who is this NOT for?

24. Rate it 1-10. Why?

25. Anything else you'd like to add?
```

### **For Video Testimonials**

```
Warm-Up Questions (get them talking):

1. How's your day going?

2. What are you working on right now?

3. How long have you been using Backend Starter Kit?

Main Questions:

4. What was your biggest pain point before?

5. What changed after using the kit?

6. What's your favorite feature?

7. How much time did it save you?

8. What would you tell a developer considering it?

Closing:

9. Any final thoughts?

10. Where can people find you/your work?
```

## 🎥 **Video Testimonial Guide**

```

• Smartphone camera (1080p)
• Quiet room
• Natural light (face window)
• Laptop for Zoom call

Better:
• USB microphone (Blue Yeti)
• Ring light
• Webcam (Logitech C920)
• Plain background

Pro:
• Professional camera
• Lapel mic
• Professional lighting
• Green screen (optional)

```
Option 1: Zoom Call (Easiest)
• Schedule 15-min call
• Record with Zoom

• User records on their time

Option 3: In-Person (Best Quality)
• Meet at co-working space

## 🎯 **Best Practices**

### **DO:**

```
✅ Ask at the right moment (after wins)
✅ Make it easy (provide templates)
✅ Offer multiple formats (text, video, call)
✅ Follow up (once or twice)
✅ Say thank you (always)
✅ Get permission before publishing
✅ Show real results (specific numbers)
✅ Include photos when possible
✅ Update testimonials regularly
```

### **DON'T:**

```
❌ Fake testimonials (ever!)
❌ Pressure or guilt-trip
❌ Ask too soon after purchase
❌ Ignore negative feedback
❌ Publish without approval
❌ Use generic questions
❌ Forget to say thanks
❌ Let testimonials get stale (older than 1 year)
```
